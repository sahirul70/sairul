E-Commerce Data Analysis Dashboard in Python

About this project

Dataset Description
The dashboard is built on an e-commerce sales dataset containing detailed transaction information. Key fields include Order Date, Region, Category, Sub-Category, Customer Segment, Sales, and Profit. This structure enables multi-dimensional analysis by time (year, month), products (category/sub-category), and customers (segment, region). The dataset provides both revenue and profit metrics, which allow performance tracking across different business areas.
Python Dash Work Process
The dashboard was developed using Python Dash and Plotly. Data was first imported with Pandas, cleaned, and transformed into monthly and category-level aggregates. Interactive filters such as a year slider and region dropdown were implemented using Dash components (dcc.Slider, dcc.Dropdown). Visuals such as line charts, bar charts, and donut charts were created with Plotly Express (px.line, px.bar, px.pie). Dash callbacks ensure dynamic updates so that all graphs respond instantly to filter changes. Styling was applied through CSS to create a dark theme and consistent visual design.
✅ Overall Summary
This dashboard provides a comprehensive overview of sales and profit trends. It highlights monthly performance, category profitability, sub-category contributions, and customer segment insights. Managers can quickly identify growth drivers, loss-making areas, and regional differences, making it a practical tool for data-driven decision-making in e-commerce strategy.

Regulatory Controls Monitoring Dashboard

About this project

Dataset Description
The dataset represents regulatory inspection and compliance control data collected across multiple agencies and sites (SITESAGENCES), analyzed by DEX (regional/functional units).
Work Process
Here’s a high-level workflow for building this dashboard:
Data Collection
Collect inspection and compliance reports from regulatory systems.
Import data from field audits, internal quality checks, and agency logs.
Extract site-wise and DEX-wise control records..
Data Transformation
Clean and standardize site and DEX names.
Derive indicators:
Passage (Qualité) = Passed Controls ÷ Total Controls
Conformité Planning = On-Time Controls ÷ Planned Controls
Levées de Réserve % = Resolved Reserves ÷ Total Reserves
Create summary tables for “Top 5/10/15 sites”
Dashboard Development
Choose BI Tool Power BI.
Build interactive filters (Date, SITESAGENCES, DEX).
KPI Cards for quick indicators.
Pie/Bar charts for distribution by status.
Narrative “Summary Section” for management.
Validation & Testing
Cross-check with compliance officers.
Ensure reserve counts match inspection records.
Validate pass rate against official regulatory statistics.
Deployment & Monitoring
Dashboard published for compliance managers.
Updated regularly (weekly/monthly).
Used in regulatory review meetings to track performance and resolve critical issues.
✅ Summary:
This Regulatory Controls Monitoring Dashboard provides a real-time compliance overview. It consolidates quality, planning, and reserve resolution data at multiple levels (DEX, sites, agencies). Management can track inspection pass rates, planning adherence, and reserve closure progress, enabling data-driven regulatory compliance decisions.

HR Analysis Dashboard

About this project

Dataset Description
The dataset combines HR data (attrition, employees) and operational performance data (delivery & preparation work). It is likely sourced from HRIS (Human Resource Information Systems), Project Management Tools, and Operational Logs.
Work Process
Here’s a high-level workflow for building this dashboard:
Data Collection
HR data: Extract from HRIS system (e.g., SAP SuccessFactors, Workday).
Operations data: Extract from task/project management tools (Power BI).
Standardize into a single data warehouse (SQL, Excel, Power BI model).
Data Transformation Clean and normalize channel names.
Clean data (normalize roles, education, regions).
Create calculated fields:
Attrition Rate = Attrited ÷ Total Employees
Active Employees = Total – Attrited
Delivery Qty = Completed Deliverables
Blocker % = Blocked Deliveries ÷ Total Deliveries
Aggregate by Department, Region, Month, Job Role.
Dashboard Development
Tool: Power BI (given style and visuals).
Page 1 (Attrition Dashboard):
KPI cards, pie/bar charts for attrition by different factors.
Page 2 (Weekly Staff Performance):
KPI cards, monthly trends, blocker analysis, region-wise breakdown.
Validation & Testing
Cross-check with HR reports (headcount, attrition).
Verify delivery numbers with operational logs.
Validate blocker definitions with project managers.
Deployment
Publish dashboard for HR & Operations leadership.
Schedule weekly refresh for operational data, monthly refresh for HR data.
Used for decision-making on staffing needs, attrition reduction, and workload planning.
✅ Summary:
This combined dashboard provides a holistic view of workforce and operations. Page 1: Focuses on HR analytics (attrition, employee demographics).
Page 2: Focuses on operational performance (delivery, blockers, demand codes).
Together, they allow management to balance workforce capacity with business delivery performance.

LLGA - Dashboard

About this project

Dataset Description
This looks like a Sales & Customer Analytics dataset combining financial, customer, and geographic data. It likely comes from ERP/CRM systems (SAP, Salesforce, Dynamics) and financial systems.
Work Process
Data Collection
Pull sales and finance data from ERP/CRM (orders, invoices, costs).
Get customer data (segments, industries, geography).
Combine shipping/logistics or loan/credit impact data.
Data Transformation .
Clean dataset (remove duplicates, standardize customer/product names).
Build derived fields:
Gross Profit = Revenue – Cost
Profit Margin % = Profit ÷ Revenue
Sales Growth % = (Current – Previous) ÷ Previous
Aggregate by:
Customer
Product category
Region
Time period (Month, Quarter)
Dashboard Development
Choose BI Tool Power BI.
Create KPI cards for executive summary.
Line charts for trend analysis.
Ranking tables for customers/products.
Maps for geographic sales.
Bar charts for comparative performance (top vs bottom customers).
Validation & Testing
Match totals with financial system reports.
Cross-check customer counts and revenue values.
Verify mapping/geography matches CRM records.
Deployment & Monitoring
Publish to Power BI.
Schedule refresh (daily/weekly depending on data).
Used by Sales Directors, Finance, and Customer Success teams to track performance and adjust strategy.
✅ Summary:
This dashboard is a comprehensive sales and customer analytics tool. It provides executive KPIs (revenue, profit, margin), detailed customer insights (top/bottom customers, trends), and geographic performance distribution. It helps leadership identify high-value customers, track financial health, and spot underperforming segments.

Marketing OKR Dashboard

About this project

Dataset Description
The dataset used here is a Marketing Performance Dataset, containing KPIs (Key Performance Indicators) across multiple channels and deal pipelines. Likely data sources include CRM systems (e.g., Salesforce, HubSpot), marketing automation tools, and ad platforms.
Work Process
Data Collection
Extract leads, deals, and financial data from CRM (HubSpot/Salesforce).
Pull cost data from marketing platforms (Google Ads, Meta, LinkedIn, etc.).
Aggregate into a single dataset (Excel, SQL, or Data Warehouse).
Data Transformation Clean and normalize channel names.
Create calculated fields:
MQL, SQL, Won definitions
Conversion rates
CPL = Expense ÷ Leads
CAC = Expense ÷ Won Deals
Targets (from marketing plan for 2025)
Time intelligence: Week-over-Week, Quarter, MTD (Month-to-Date).
Dashboard Development
Choose BI Tool Power BI.
Add KPI cards for quick insights.
Visualize trends by channel and deal type.
Validation & Testing
Cross-check metrics with CRM reports.
Validate conversion funnel logic.
Deployment & Monitoring
Publish dashboard.
Schedule automatic data refresh (daily/weekly).
Use dashboard for quarterly OKR reviews..
✅ Summary:
This Marketing OKR Dashboard integrates leads, deals, conversions, and cost metrics into one view. It allows management to track performance vs. goals, analyze channel efficiency (CPL, CAC), and ensure marketing activities are aligned with 2025 targets.
